# Millrace v0.1.0 Autonomous Build Proof

This document records the evidence that the Python Millrace runtime autonomously
built the Rust `millrace-ai` v0.1.0 parity implementation.

The word "build" here means the autonomous implementation campaign: Millrace
accepted seeded ideas, decomposed them into specs and tasks, executed Codex-backed
runtime stages, routed fixes and arbiter remediation, wrote the Rust crate, and
left the workspace with all parity slices completed. Human/operator work still
handled repository administration, release tagging, publishing, pausing/resuming,
and independent post-run checks.

## Closure and Remediation Evidence

Millrace did not simply generate code and stop. The arbiter plane produced
closure targets and verdicts for all eight seeded slices. Three arbiter passes
returned `REMEDIATION_NEEDED`; the runtime then converted those gaps into
follow-up incidents/specs/tasks and eventually resolved the relevant closure
work.

Important remediation surfaces:

- Slice 1: stage-result parity remediation.
- Slice 4: queue repair-lineage and workspace-boundary remediation.
- Slice 7: real adapter smoke-surface remediation.

## Interpretation

This artifact demonstrates that Python Millrace v0.16.1 could run a large,
multi-slice autonomous implementation campaign against an initially skeletal
Rust crate and produce a released Rust parity runtime.

The strongest signals are:

- Millrace decomposed 8 human-seeded ideas into 11 completed specs and 57
  completed tasks.
- It executed 261 recorded Codex-backed stage calls over 99 runs.
- It routed normal build/check/update loops, fix-needed loops, double-checks,
  arbiter closure checks, remediation planning, and deferred learning requests.
- It produced a v0.1.0 release commit spanning 193 files and 87,992 insertions
  from the claimed-crate baseline.
- It left the runtime queues empty for execution and planning, with all seeded
  parity slices represented by closure artifacts and done work.
- The separately installed `millrace-ai v0.1.0` crate later passed a real
  daemon smoke by completing useful filesystem work through real Codex-backed
  builder/checker/updater stages.

Residual caveats:

- Token totals are derived from 254 raw `turn.completed` payloads; 7 very short
  blocked/failure-path stage results had no usage payload and are not included
  in token totals.
- The campaign proves the Python runtime's ability to build the Rust parity
  implementation. It does not claim the Rust crate independently recreated this
  whole campaign without the Python daemon.
- Publishing and release tagging were operator actions after the autonomous
  build campaign completed.
